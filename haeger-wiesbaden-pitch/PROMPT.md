# Auftrag: Standalone Landingpage - Christian Haeger Juwelier Wiesbaden

## Kontext

Ich bin Adrian von FlowState, einer Webdesign-Agentur aus Wiesbaden. Ich will dem Juwelier "Christian Haeger" (HAEGER GmbH) einen Pitch machen. Die haben 12 Standorte deutschlandweit, darunter einen in Wiesbaden. Ihre aktuelle Website (christian-haeger.com) sieht zwar aufwendig aus, hat aber massive Performance-Probleme (lange Ladezeiten, schlechte Mobile-Kompatibilitaet, die Standort-Unterseiten sehen broken aus).

Mein Angebot: Ich baue ihnen die Standortseiten als separate, schnelle Standalone-Landingpages die unabhaengig vom Hauptshop laufen. Als Pitch-Material baue ich erstmal die Wiesbaden-Seite als Prototyp.

## Was du bauen sollst

Eine einzelne, selbststaendige HTML-Landingpage fuer den Standort **Wiesbaden** von Christian Haeger. Alles in einer Datei (HTML + inline CSS + inline JS), keine externen Abhaengigkeiten ausser Google Fonts.

## Design-Anforderungen

- **Luxus-Aesthetic**: Das ist ein Juwelier der Rolex, Cartier etc. verkauft. Das Design muss premium wirken - denke dunkle Toene, Gold-Akzente, viel Whitespace, elegante Typografie
- **Mobile-First**: Muss auf dem Handy perfekt aussehen, das ist der Hauptkritikpunkt an ihrer aktuellen Seite
- **Extrem schnell ladend**: Kein Bloat, kein unnoetige JS-Frameworks, lean und schnell. Das ist das Kernargument meines Pitches
- **Moderne Animationen**: Dezente, elegante Scroll-Animationen und Hover-Effekte. Nichts uebertriebenes, aber es soll sich hochwertig anfuehlen
- **Kein Dummy-Content**: Nutze ausschliesslich die echten Daten unten

## Technische Anforderungen

- Eine einzige index.html Datei
- Inline CSS und JS (keine externen Dateien ausser Google Fonts)
- Semantic HTML5
- CSS Custom Properties fuer Farben/Fonts
- Smooth Scroll Navigation
- Intersection Observer fuer Scroll-Animationen
- Responsive Breakpoints: Mobile (< 768px), Tablet (768-1024px), Desktop (> 1024px)
- Kein jQuery, kein Bootstrap, kein Framework - pures HTML/CSS/JS
